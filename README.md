## Original Development Context

These materials were originally developed to support a Microsoft 365 
Copilot implementation training program for a regulated nonprofit 
public-sector entity operating electric generation and transmission 
infrastructure in Texas — an environment governed by NERC CIP 
cybersecurity standards, Texas TAC Chapter 202, and state procurement 
compliance requirements.

The training program addressed the specific challenges of deploying AI 
tools in high-compliance environments: data classification controls, 
tenant isolation requirements, acceptable use boundaries for operational 
technology (OT) adjacent systems, and workforce adoption in a 
security-conscious culture.

Materials have been generalized (organizational identifiers removed) and 
updated for broader use, including submission as portfolio artifacts for 
Texas DIR contractor evaluation.

## Regulatory Frameworks Covered

- **Texas TDPSA** — Texas Data Privacy and Security Act (effective July 1, 2024)
- **NERC CIP** — Critical Infrastructure Protection standards; CIP-005/007/010/011/013 applicability to AI/cloud tools
- **EU AI Act** — Implementation timeline; deployer obligations (full enforcement August 2, 2026)
- **GDPR** — Data subject rights and processing obligations
- **HIPAA / Texas TAC 202** — Healthcare and state agency data handling
- **NIST AI RMF 1.0** — Voluntary AI risk management framework
- **ISO/IEC 42001:2023** — Certifiable AI management system standard
- **Microsoft Purview** — Sensitivity labels, DLP, DSPM for AI, audit logging
